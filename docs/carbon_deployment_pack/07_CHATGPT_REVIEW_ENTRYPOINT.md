# Eagle Eye ChatGPT Reviewer Entrypoint

This document gives a stable, non-JS review surface for external agents (including ChatGPT-like reviewers) to inspect Eagle Eye behavior without relying on live browser rendering.

## Stable Artifact URLs (GitHub Pages)

Replace `<owner>` and `<repo>` with your GitHub repository values:

- Latest machine-readable review JSON:  
  `https://<owner>.github.io/<repo>/review/latest/review_index.json`
- Latest human-readable summary:  
  `https://<owner>.github.io/<repo>/review/latest/review_summary.md`
- Latest screenshots folder:  
  `https://<owner>.github.io/<repo>/review/latest/screenshots/`
- Review landing page:  
  `https://<owner>.github.io/<repo>/review/`

These artifacts are generated by the UI automation pipeline and published from CI.

## Artifact Contract (`review_index.json`)

Top-level fields:

- `run_id`
- `timestamp_utc`
- `git_sha`
- `base_url`
- `api_base_url`
- `overall_status`
- `totals`
- `scenarios[]`
- `api_checks[]`

Per scenario:

- `id`
- `category`
- `query`
- `status` (`pass` or `fail`)
- `error_code` (for failures, e.g. `render_timeout`, `query_submit_failed`, `section_missing`, `validation_failed`)
- `error_message`
- `checks[]` (assertion-level pass/fail)
- `extracted` (key UI fields parsed from rendered text)
- `screenshots[]`

## Reviewer Interpretation Rules

1. Treat `overall_status=pass` as the default green signal.
2. If `overall_status=fail`, inspect:
   - failed scenario IDs,
   - `error_code`,
   - screenshot for that scenario.
3. For carbon scenarios, use `extracted.carbon_result_state` to separate:
   - deterministic computed outputs (`COMPUTED`, `COMPUTED_ZERO`),
   - non-computable/retrieval-only outputs (`NOT_COMPUTABLE`, `RETRIEVAL_ONLY`).
4. Use `api_checks` to confirm backend health and route-level availability independent of UI rendering.

## Known Limits (Explicit)

- Public live app URLs can be tunnel/session dependent.
- The review artifact URL is the stable source for external analysis.
- Scenario outcomes are data-dependent; fixture queries are selected to minimize false negatives across common runtime datasets.

## Local fallback generation (same contract)

From repository root:

```bash
./run_ui_review.sh
```

This writes:

- `/Users/praharshchintu/Documents/New project/review/latest/review_index.json`
- `/Users/praharshchintu/Documents/New project/review/latest/review_summary.md`
- `/Users/praharshchintu/Documents/New project/review/latest/screenshots/*`

and archives the run to:

- `/Users/praharshchintu/Documents/New project/review/runs/<run_id>/`
